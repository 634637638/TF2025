# 一键智能导入 - 快速使用指南

## 🚀 5分钟快速上手

### 第一步：访问页面
```
浏览器打开: http://localhost:5176/data-optimization
点击 "远程数据同步" 标签
```

### 第二步：连接数据库
填写外部数据库信息：
```
主机地址: 例如 192.168.1.100
端口: 3306（默认）
用户名: 数据库用户名
密码: 数据库密码
数据库名: 源数据库名称
```

点击 "连接数据库" 按钮

### 第三步：一键智能导入 ✨

连接成功后，页面顶部会显示紫色渐变的横幅：

```
┌────────────────────────────────────────────────────┐
│  🪄  一键智能导入                                    │
│                                                     │
│  系统将自动识别所有表并智能匹配，无需手动配置        │
│                                                     │
│  ✅ 自动识别表类型    ✅ 智能匹配字段映射            │
│  ✅ 自动创建关联数据  ✅ 批量同步执行                │
│                                                     │
│           [ 🪄 一键智能导入 ]                       │
└────────────────────────────────────────────────────┘
```

点击 **"🪄 一键智能导入"** 按钮

### 第四步：等待完成

系统会自动执行以下操作：

1. **扫描表** - 识别外部数据库的所有表
2. **识别类型** - 智能判断表类型（手机、客户、销售等）
3. **生成映射** - 自动创建字段映射关系
4. **执行同步** - 批量同步数据到本地
5. **显示结果** - 展示同步统计信息

### 第五步：查看结果

同步完成后，会显示详细的统计信息：

```

识别的表:
  old_phones → phones (插入 100 条, 更新 50 条)
  old_customers → customers (插入 200 条, 更新 30 条)
  old_brands → brands (插入 10 条)

总计:
  插入: 310 条

  跳过: 0 条
```

## ⚠️ 注意事项

1. **首次使用建议先备份**
   ```sql
   -- 备份重要表
   CREATE TABLE phones_backup AS SELECT * FROM phones;
   ```

2. **确保网络连接稳定**
   - 大数据量同步时保持网络稳定
   - 避免中断操作

3. **检查表结构兼容性**
   - 源表和目标表字段应尽量匹配
   - 使用"预检查"功能验证

## 🔧 高级功能

### 手动配置同步

如果需要更精确的控制，可以使用手动配置：

1. 点击 "下一步" 进入向导
2. 手动选择源表和目标表
3. 配置字段映射关系
4. 选择同步模式：

   - **update** - 只更新已有数据
   - **upsert** - 插入或更新（推荐）
   - **replace** - 清空后重新导入
5. 执行同步

## 💡 常见问题

### Q1: 连接失败怎么办？
**A**: 检查以下几点：
- 数据库地址是否正确
- 用户名和密码是否正确
- 数据库用户是否有足够权限
- 防火墙是否允许连接

### Q2: 识别表类型失败？
**A**:
- 检查表名是否符合规范（如：phones, customers）
- 检查字段名是否包含关键信息（如：imei, phone）
- 使用手动配置方式指定映射关系

### Q3: 同步速度慢？
**A**:
- 调整批量大小（默认100条/批）
- 使用索引字段作为匹配字段
- 分批同步大量数据

### Q4: 如何查看创建的关联数据？
**A**: 同步完成后，可以在相应表中查看：
```sql
-- 查看自动创建的品牌
SELECT * FROM brands ORDER BY id DESC LIMIT 10;

-- 查看自动创建的客户
SELECT * FROM customers ORDER BY id DESC LIMIT 10;
```
