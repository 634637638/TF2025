# 远程数据同步功能使用说明

## 功能概述

远程数据同步功能允许您直接连接其他MySQL数据库，自动匹配数据并执行同步操作。系统会自动创建关联的基础数据（品牌、型号、颜色等），无需手动准备。

## ✨ 核心特性

### 1. 自动创建基础数据
当同步数据时，系统会自动创建以下关联数据：
- ✅ **品牌** (brands) - 根据品牌名称自动创建
- ✅ **型号** (models) - 根据型号名称自动创建
- ✅ **颜色** (colors) - 根据颜色名称自动创建
- ✅ **内存** (memories) - 根据内存大小自动创建
- ✅ **供应商** (suppliers) - 根据供应商名称自动创建
- ✅ **店铺** (stores) - 根据店铺名称自动创建
- ✅ **客户** (customers) - 根据手机号自动创建，并生成会员号
- ✅ **操作员** (users) - 根据姓名自动创建

### 2. 智能字段匹配
- 自动识别字段名称
- 智能匹配相似字段
- 支持自定义映射关系

### 3. 四种同步模式
| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **insert** | 只插入新数据 | 初始化导入 |
| **update** | 只更新已存在数据 | 定期更新 |
| **upsert** | 存在则更新，不存在则插入 | 日常同步（推荐） |
| **replace** | 清空后重新导入 | 完全替换数据 |

### 4. 数据匹配规则
- 支持单个或多个匹配字段
- 根据键字段值判断数据是否存在
- 示例：`phone`、`imei`、`serial_number`

## 📖 使用步骤

### 第一步：访问功能
1. 登录系统
2. 访问 `http://localhost:5176/data-optimization`
3. 点击"远程数据同步"标签

### 第二步：连接外部数据库
1. 填写数据库连接信息：
   - **主机地址**: 数据库服务器IP
   - **端口**: MySQL端口（默认3306）
   - **用户名**: 数据库用户名
   - **密码**: 数据库密码
   - **数据库名**: 要连接的数据库名称
2. 点击"连接数据库"
3. 连接成功后会显示在已连接列表

### 第三步：选择表
1. **左侧列表**: 选择外部数据库的源表
2. **右侧列表**: 选择本地数据库的目标表
3. 使用搜索框快速查找表名

### 第四步：配置字段映射
1. 系统自动显示源表的所有字段
2. 点击"智能匹配"按钮，系统自动建议映射关系
3. 手动调整映射关系（可选）
4. 配置同步选项：
   - **同步模式**: 选择数据处理方式
   - **匹配字段**: 选择用于判断数据是否存在的字段
   - **批量大小**: 每批次处理的数据量（建议100）

### 第五步：数据预检查
1. 点击"开始检查"按钮
2. 系统分析并显示：
   - 总数据量
   - 新数据数量（需要插入）
   - 需要更新的数据数量
   - 匹配示例
3. 确认无误后继续

### 第六步：执行同步
1. 点击"开始同步"按钮
2. 实时查看同步进度：
   - 已插入数量
   - 已更新数量
   - 已跳过数量
   - 失败数量
3. 同步完成后查看详细统计

## 💡 使用示例

### 示例1：同步手机数据（自动创建关联数据）

**场景**: 从老系统同步手机数据到新系统

**配置**:
```
源表: old_phones
目标表: phones
同步模式: upsert
匹配字段: imei
```

**字段映射**:
```
imei → imei
serial_number → serial_number
brand_name → brand_id (自动创建品牌)
model_name → model_id (自动创建型号)
color_name → color_id (自动创建颜色)
memory_size → memory_id (自动创建内存)
supplier_name → supplier_id (自动创建供应商)
store_name → store_id (自动创建店铺)
```

**执行结果**:
- ✅ 自动创建品牌: 苹果、华为、小米等
- ✅ 自动创建型号: iPhone 15、Mate 60等
- ✅ 自动创建颜色: 黑色、白色等
- ✅ 自动创建内存: 256GB、512GB等
- ✅ 同步手机数据: 插入100条，更新50条

### 示例2：同步客户数据（自动生成会员号）

**场景**: 从外部系统同步客户信息

**配置**:
```
源表: external_customers
目标表: customers
同步模式: upsert
匹配字段: phone
```

**字段映射**:
```
name → name
phone → phone
email → email
```

**执行结果**:
- ✅ 自动创建客户记录
- ✅ 自动生成会员号: TF10000, TF10001等
- ✅ 同步客户数据: 插入200条，更新30条

## 🔧 技术说明

### 自动创建数据的触发条件
系统会在以下情况下自动创建基础数据：

1. **品牌 (brands)**
   - 当 `brand_id` 字段值是字符串（品牌名称）时
   - 检查数据库是否存在该品牌
   - 不存在则自动创建

2. **型号 (models)**
   - 当 `model_id` 字段值是字符串（型号名称）时
   - 检查数据库是否存在该型号
   - 不存在则自动创建

3. **颜色 (colors)**
   - 当 `color_id` 字段值是字符串（颜色名称）时
   - 检查数据库是否存在该颜色
   - 不存在则自动创建

4. **内存 (memories)**
   - 当 `memory_id` 字段值是字符串（内存大小）时
   - 检查数据库是否存在该内存规格
   - 不存在则自动创建

5. **供应商 (suppliers)**
   - 当 `supplier_id` 字段值是字符串（供应商名称）时
   - 检查数据库是否存在该供应商
   - 不存在则自动创建

6. **店铺 (stores)**
   - 当 `store_id` 字段值是字符串（店铺名称）时
   - 检查数据库是否存在该店铺
   - 不存在则自动创建

7. **客户 (customers)**
   - 当 `customer_id` 字段值是字符串（手机号）时
   - 检查数据库是否存在该手机号
   - 不存在则自动创建，并生成会员号

8. **操作员 (users)**
   - 当 `inventory_operator_id` 或 `sale_operator_id` 字段值是字符串时
   - 检查数据库是否存在该操作员
   - 不存在则自动创建

### 数据映射机制
系统使用内存映射（Map）来跟踪已创建的数据：
- 避免重复查询数据库
- 提高性能
- 确保一致性

### 性能优化
1. **批量处理**: 默认每批处理100条数据
2. **事务保护**: 每批次使用事务，确保数据一致性
3. **内存映射**: 缓存已创建的数据ID
4. **进度跟踪**: 实时显示同步进度

## ⚠️ 注意事项

### 安全建议
1. 使用只读权限的数据库账号连接源数据库
2. 同步前先备份目标数据库
3. 在测试环境验证后再在生产环境使用
4. 定期检查同步日志

### 数据一致性
1. 确保源数据和目标数据结构兼容
2. 使用"预检查"功能验证配置
3. 检查字段映射是否正确
4. 验证匹配字段是否合理

### 性能建议
1. 大数据量同步时，适当增加批量大小
2. 使用索引字段作为匹配字段
3. 避免在高峰期执行同步
4. 分批次同步大量数据

## 🚀 高级用法

### 定时同步
使用自动化脚本实现定时同步：
```bash
# 每天凌晨2点执行同步
0 2 * * * cd /path/to/backend && node auto-sync-database.js
```

### 多源同步
从多个数据源同步数据：
```bash
# 同步主数据库
node auto-sync-database.js config-master.json

# 同步分店数据库
node auto-sync-database.js config-branch1.json
```

### 条件同步
修改脚本，添加WHERE条件过滤数据：
```javascript
// 只同步最近30天的数据
const [sourceData] = await sourceConnection.query(
  `SELECT * FROM ${sourceTable} WHERE created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)`
);
```
