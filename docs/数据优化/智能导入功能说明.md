# 智能导入功能说明

## 概述

智能导入是一项新增的数据导入策略，它会自动分析云端数据和本地Excel数据的状态差异，智能选择最佳的处理方式。

---

## 导入策略对比

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **🌟 智能导入** | AI自动判断最佳处理方式 | **推荐使用**，适合大多数场景 |
| **跳过重复** | 保留现有数据，只导入新数据 | 只想添加全新手机 |
| **覆盖重复** | 完全覆盖旧记录 | Excel是完整准确的数据 |
| **合并重复** | 只更新非空字段 | 保留云端数据，补充新信息 |
| **完全替换** | 删除所有数据重新导入 | 初始化系统 |

---

## 智能导入的四种场景

### 场景 1: 📦 → ✨ 库存 → 已售

**情况**：云端显示库存，本地Excel显示已售

**处理方式**：
- ✅ **使用 overwrite 逻辑，以本地Excel为准完整更新所有字段**
- ✅ 更新 `phones.status` 为 `sold`
- ✅ 更新 `salestime` 为销售日期
- ✅ 更新品牌、型号、颜色、内存、供应商、店铺等所有字段
- ✅ 更新销售价格、销售员、入库员等信息
- ✅ 创建/更新 `sales` 表记录
- ✅ 自动创建客户、销售员等缺失的基础数据

**关键特性**：
- **完整覆盖**：所有字段按本地Excel更新，包括空值
- **自动创建**：自动创建品牌、型号、颜色、客户、销售员等缺失数据
- **智能匹配**：品牌、型号等支持模糊匹配

**日志输出**：
```
🧠 [智能] 云端状态=库存, Excel状态=已售
✨ [智能] 场景1: 库存→已售，以本地Excel为准完整更新所有数据
```

---

### 场景 2: ✨ → 📦 已售 → 库存

**情况**：云端显示已售，本地Excel显示库存

**处理方式**：
- 🛡️ **保持云端的已售状态**（不降级）
- ❌ 不更新 phones 记录
- ✅ 可能更新客户 Apple ID

**原因**：一旦手机售出，不应该退回到库存状态

**日志输出**：
```
🧠 [智能] 云端状态=已售, Excel状态=库存
🛡️ [智能] 场景2: 已售→库存，保持已售状态（不降级）
```

---

### 场景 3: ✨ → 🔄 已售 → 已售

**情况**：云端和本地Excel都显示已售

**处理方式**：
- ✅ **使用 overwrite 逻辑，以本地Excel为准完整更新所有字段**
- ✅ 更新所有销售信息（客户、销售日期、销售员、销售价格等）
- ✅ 更新品牌、型号、颜色、内存等所有字段
- ✅ 自动创建/更新客户、销售员等基础数据

**日志输出**：
```
🧠 [智能] 云端状态=已售, Excel状态=已售
🔄 [智能] 场景3: 已售→已售，以本地Excel为准完整更新销售信息
```

---

### 场景 4: 📦 → 📦 库存 → 库存

**情况**：云端和本地Excel都显示库存

**处理方式**：
- 📦 **保持库存状态不变**
- ❌ 不更新 phones 记录

**日志输出**：
```
🧠 [智能] 云端状态=库存, Excel状态=库存
📦 [智能] 场景4: 库存→库存，保持库存状态
```

## 智能销售状态判断

智能导入会从多个维度判断Excel中的销售状态：

| 字段 | 权重 | 说明 |
|------|------|------|
| `状态` = "已销售" | ⭐⭐⭐⭐⭐ | 明确标记为已售 |
| `销售日期` 有值 | ⭐⭐⭐⭐⭐ | 有销售日期说明已售 |
| `客户姓名` + `手机号` | ⭐⭐⭐⭐ | 有客户信息通常已售 |
| `销售员` 有值 | ⭐⭐⭐ | 有销售员通常已售 |

**判断逻辑**：
```javascript
const isExcelSold = hasStatusSold || hasSaleDate || (hasCustomerInfo && hasPhone);
```

## 后端实现

### 关键代码位置

| 功能 | 文件 | 行号 |
|------|------|------|
| 智能策略判断 | `data-import.service.js` | 636-672 |
| 状态检测 | `data-import.service.js` | 617-634 |
| 分析返回云端状态 | `data-import.service.js` | 290-297 |

### 核心逻辑

```javascript
// 判断云端状态
const isCloudSold = !!duplicateRecord.saleId;

// 判断Excel状态
const isExcelSold = hasStatusSold || hasSaleDate || (hasCustomerInfo && hasPhone);

// 智能分支
if (strategy === 'smart') {
  if (!isCloudSold && isExcelSold) {
    // 库存→已售：使用 overwrite 完整更新
    strategy = 'overwrite';
    row['状态'] = '已销售';
  } else if (isCloudSold && !isExcelSold) {
    // 已售→库存：保持
    return { status: 'skipped', reason: '云端已售，保持不变' };
  } else if (isCloudSold && isExcelSold) {
    // 已售→已售：使用 overwrite 完整更新
    strategy = 'overwrite';
  } else {
    // 库存→库存：保持
    return { status: 'skipped', reason: '双方都是库存，保持不变' };
  }
}
```

## 使用建议

### 推荐使用场景

1. **日常销售数据同步**：本地Excel是最新的销售记录，需要更新云端状态
2. **多店铺数据汇总**：各店铺的Excel数据需要合并到云端
3. **定期数据更新**：每天/每周将新销售数据导入系统

### 不推荐使用场景

1. **首次导入数据**：使用"完全替换"或直接导入新数据
2. **完全重建数据**：使用"完全替换"
3. **只添加新手机**：使用"跳过重复"

**导入结果**：
```

✅ phones.salestime → '2025-01-10'
✅ phones.sale_price → 4999
✅ phones.brand_id → 品牌ID（自动创建或匹配）
✅ phones.model_id → 型号ID（自动创建或匹配）
✅ phones.color_id → 颜色ID（自动创建或匹配）

✅ phones.supplier_id → 供应商ID（自动创建或匹配）
✅ phones.store_id → 店铺ID（自动创建或匹配）
```
