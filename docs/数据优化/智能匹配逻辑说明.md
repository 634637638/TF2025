# 智能匹配逻辑说明

## 概述

智能导入功能在处理数据时，会**优先匹配数据库中已有的数据**，只有匹配不到时才会创建新数据。这样可以避免重复创建品牌、型号、客户等基础数据。

---

## 匹配优先级

### 1. 内存映射缓存
```
第一次查询后，结果会缓存到内存 Map 中
后续相同的查询直接从内存返回，避免重复查询数据库
```

### 2. 数据库查询
```
先查询数据库，检查是否已存在匹配的记录
只有数据库中也不存在时，才创建新记录
```

### 3. 智能创建
```
只有在以上两者都找不到时，才创建新数据
```

## 各类数据的匹配规则

### 品牌 (brands)

**匹配字段**: `name`

**匹配策略**:
1. **精确匹配**: 名称完全相同
2. **模糊匹配**: 忽略大小写、空格、特殊字符
   - `iPhone 12` ≈ `iphone12`
   - `Apple` ≈ `apple`
3. **包含匹配**: 处理缩写或简称
   - `华为` ∋ `华为荣耀`

**代码示例**:
```javascript
// 精确匹配
SELECT id FROM brands WHERE name = 'Apple' AND status = 1

// 模糊匹配（忽略格式差异）
SELECT id FROM brands WHERE name = 'apple' AND status = 1

// 包含匹配
SELECT id FROM brands WHERE name LIKE '%Apple%' AND status = 1
```

### 客户 (customers)

**匹配字段**: `phone` (手机号)

**匹配策略**:
1. **精确匹配**: 手机号完全相同
2. **唯一性**: 手机号是客户的唯一标识

**特殊处理**:
- 匹配到已有客户时，会更新 Apple ID（如果提供）
- 未找到时，自动创建新客户，会员号从 TF10000 开始递增

**代码示例**:
```javascript
// 1. 检查内存映射
if (map.has(phone)) {
  return map.get(phone);
}

// 2. 查询数据库
const [existingCustomers] = await connection.execute(
  'SELECT id, name, phone, apple_id FROM customers WHERE phone = ? LIMIT 1',
  [phone]
);

if (existingCustomers.length > 0) {
  // 找到已有客户，添加到映射
  map.set(phone, existingCustomers[0].id);
  return existingCustomers[0].id;
}

// 3. 未找到，创建新客户
const [result] = await connection.execute(
  'INSERT INTO customers (name, phone, member_number, ...)'
);
```

### 员工/销售员 (users)

**匹配字段**: `name` 或 `username`

**匹配策略**:
1. **精确匹配**: 姓名或用户名完全相同
2. **多字段匹配**: 同时检查 `name` 和 `username`
   - `name = '小王'` OR `username = '小王'`

**特殊处理**:
- 未找到时，自动创建新员工
- 默认密码: `123456`
- 默认角色: `employee`
- 用户名自动生成（中文转拼音首字母，或使用小写）

**代码示例**:
```javascript
// 1. 检查内存映射
if (map.has(name)) {
  return map.get(name);
}

// 2. 查询数据库
const [existingUsers] = await connection.execute(
  'SELECT id, name, username FROM users WHERE name = ? OR username = ?',
  [name, name]
);

if (existingUsers.length > 0) {
  // 找到已有员工
  map.set(name, existingUsers[0].id);
  return existingUsers[0].id;
}

// 3. 未找到，创建新员工
const [result] = await connection.execute(
  'INSERT INTO users (username, name, password, role_id, status, created_at)'
);
```

## 智能匹配代码实现

### 核心方法: `getOrCreate()`

**文件**: `backend/src/services/data-import.service.js`

**位置**: [data-import.service.js:1138-1200](backend/src/services/data-import.service.js#L1138-L1200)

**流程图**:
```
┌─────────────────────────────────────────────────────────┐
│ getOrCreate(connection, table, name, map, field, extra) │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
              ┌───────────────────────┐
              │ 检查内存映射 map.has() │
              └───────────────────────┘
                    │           │
                   有           无
                    │           │
                    ▼           ▼
              ┌─────────┐   ┌──────────────────┐
              │返回缓存ID│   │ 查询数据库        │
              └─────────┘   │ findExisting()   │
                             └──────────────────┘
                                   │           │
                                  找到         未找到
                                   │           │
                                   ▼           ▼
                             ┌─────────┐   ┌────────────┐
                             │缓存并返回│   │ 创建新记录  │
                             └─────────┘   │ insert新数据│
                                           └────────────┘
                                                │
                                                ▼
                                          ┌────────────┐
                                          │ 缓存并返回  │
                                          └────────────┘
```

**代码**:
```javascript
async getOrCreate(connection, table, name, map, field = 'name', extraData = {}) {
  if (!name) return null;

  const normalizedName = String(name).trim();
  if (!normalizedName) return null;

  // 1. 检查内存映射
  if (map.has(normalizedName)) {
    return map.get(normalizedName);
  }

  // 2. 智能匹配数据库
  const existingRecords = await this.findExistingRecord(connection, table, field, normalizedName);

  if (existingRecords && existingRecords.length > 0) {
    const id = existingRecords[0].id;
    map.set(normalizedName, id);
    console.log(`  ✓ 匹配到${table}: "${normalizedName}" → ID ${id}`);
    return id;
  }

  // 3. 未找到，创建新记录
  console.log(`  ➕ 创建新${table}: "${normalizedName}"`);
  const [result] = await connection.execute(
    `INSERT INTO ${table} (${field}, status) VALUES (?, 1)`,
    [normalizedName]
  );
  map.set(normalizedName, result.insertId);
  return result.insertId;
}
```

### 智能查询方法: `findExistingRecord()`

**文件**: `backend/src/services/data-import.service.js`

**位置**: [data-import.service.js:1201-1236](backend/src/services/data-import.service.js#L1201-L1236)

**匹配层次**:
1. **精确匹配**: `WHERE field = ?`
2. **模糊匹配**: 忽略大小写、空格、特殊字符
3. **包含匹配**: 处理简称或缩写

**代码**:
```javascript
async findExistingRecord(connection, table, field, name) {
  // 1. 精确匹配
  const [exactMatches] = await connection.execute(
    `SELECT id, ${field} FROM ${table} WHERE ${field} = ? AND status = 1 LIMIT 1`,
    [name]
  );

  if (exactMatches.length > 0) {
    return exactMatches;
  }

  // 2. 模糊匹配（忽略大小写、空格、特殊字符）
  const normalizedName = name.toLowerCase().replace(/[\s\-_]/g, '');

  const [allRecords] = await connection.execute(
    `SELECT id, ${field} FROM ${table} WHERE status = 1`
  );

  for (const record of allRecords) {
    const recordName = String(record[field] || '').toLowerCase().replace(/[\s\-_]/g, '');

    // 完全匹配（忽略格式差异）
    if (recordName === normalizedName) {
      console.log(`  🔍 模糊匹配${table}: "${record[field]}" ≈ "${name}"`);
      return [record];
    }

    // 包含匹配（处理缩写）
    if (recordName.includes(normalizedName) || normalizedName.includes(recordName)) {
      console.log(`  🔍 包含匹配${table}: "${record[field]}" ∋ "${name}"`);
      return [record];
    }
  }

  return null;
}
```

## 性能优化

### 内存映射缓存
- 同一个导入任务中，相同数据只查询一次数据库
- 结果缓存到 Map 中，后续直接从内存返回
- 大幅减少数据库查询次数

### 批量预加载
- 在导入开始前，会先批量加载所有基础数据
- 构建完整的数据映射表
- 后续匹配直接在内存中进行

---

## 总结

**智能匹配 = 先匹配 + 后创建**

1. ✅ **优先使用已有数据**：避免重复创建
2. ✅ **智能模糊匹配**：兼容各种格式差异
3. ✅ **内存缓存优化**：提升性能
4. ✅ **自动创建缺失数据**：确保导入成功
