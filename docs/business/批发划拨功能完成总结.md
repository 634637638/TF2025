# 批发/代划拨功能完成总结

## ✅ 已完成的工作

### 1. 数据库迁移 ✅
**文件**: `/backend/database/migrations/add_peer_transfer_fields_v2.sql`

**新增表**:
- `wholesale_records` - 批发/划拨记录表
- `proxy_settlements` - 代划拨供应商结算表

**修改表**:
- `phones` - 添加批发/划拨相关字段
  - `business_type` ENUM('retail', 'wholesale', 'supplier_proxy', 'supplier_return')
  - `wholesale_date`, `wholesale_customer_id`, `wholesale_price`
  - `proxy_supplier_id`, `proxy_settled`
- `customers` - 添加客户类型字段
  - `customer_type` ENUM('retail', 'wholesale', 'both')

**状态扩展**:
- `peer_transfer` - 同行调货（批发）
- `supplier_proxy` - 代供应商划拨
- `returned` - 已退货

### 2. 后端API ✅

**控制器**: `/backend/src/controllers/transfer.controller.js`
- `wholesaleToPeer()` - 批发给同行
- `proxyTransferForSupplier()` - 代供应商划拨
- `getTransferRecords()` - 获取记录列表
- `getTransferStatistics()` - 获取统计数据

**服务层**: `/backend/src/services/transfer.service.js`
- 支持单台/多台操作
- 自动计算利润（批发模式）
- **批发模式自动创建销售记录**（包含 store_id 和 salesperson_name）
- 代划拨模式不创建销售记录（不产生货款）
- 事务处理确保数据一致性

**路由**: `/backend/src/routes/transfers.js`
- `POST /api/transfers/wholesale` - 批发给同行
- `POST /api/transfers/proxy` - 代供应商划拨
- `GET /api/transfers/records` - 获取记录
- `GET /api/transfers/statistics` - 获取统计

### 3. 前端组件 ✅

**批发/划拨对话框**: `/frontend/src/components/WholesaleModal.vue`
- 支持两种模式切换
- 客户/供应商选择

- 响应式设计

**销售页面集成**: `/frontend/src/views/sales/SalesView.vue`

- 支持单台/多台选择

**综合查询更新**:
- 后端状态映射更新

## 📊 两种业务场景对比

| 特性 | 批发给同行 | 代供应商划拨 |
|------|-----------|-------------|
| **使用场景** | 自己批发给同行赚钱 | 代供应商把手机划拨给其客户 |
| **是否产生货款** | ✅ 是（计入供应商应付） | ❌ 否（不计入） |
| **是否计算利润** | ✅ 是（批发价 - 进价） | ❌ 否（仅记录） |
| **创建销售记录** | ✅ 是（自动创建） | ❌ 否 |
| **business_type** | `wholesale` | `supplier_proxy` |
| **手机状态** | `peer_transfer` | `supplier_proxy` |
| **示例** | 进价5000，批发5050，利润50 | 进价5000，划拨5000（代供应商） |

## 🎯 使用流程

### 场景1：批发给同行
```
1. 进入销售管理页面
2. 通过IMEI检索或列表选择手机
3. 勾选一台或多台手机
4. 点击"批发给同行"按钮
5. 选择客户（同行类型）
6. 选择销售门店和销售员
7. 输入批发价格（可修改入库价格）
8. 确认批发
9. 系统：
   - 手机状态变为"同行调货"
   - 产生货款记录
   - 计算利润
   - 自动创建销售记录
```

### 场景2：代供应商划拨
```
1. 进入销售管理页面
2. 通过IMEI检索或列表选择手机
3. 勾选一台或多台手机
4. 点击"代划拨"按钮
5. 选择供应商
6. 选择客户（供应商的客户，可选）
7. 输入划拨价格（按供应商要求）
8. 确认划拨
9. 系统：
   - 手机状态变为"代划拨"
   - 不产生货款
   - 不计算利润
```

### 综合查询检索
```
1. 进入综合查询页面
2. 状态筛选选择：
   - "同行调货" - 查看所有批发记录
   - "代划拨" - 查看所有划拨记录
3. 可以按日期、客户、供应商筛选
4. 查看详细记录和利润信息
```
