# 休假申请实时计算说明

## 核心逻辑

**系统会实时计算所有记录（包括待审批）**，确保用户不会超额申请带薪休假。

## 计算规则

### 统计范围
- ✅ 包括：`pending`（待审批）+ `approved`（已通过）
- ❌ 不包括：`rejected`（已拒绝）

### 计算公式
```
已用天数 = SUM(所有 pending + approved 状态的休假天数)
可用天数 = 本月配额 - 已用天数
```

## 场景示例

### 场景1：连续申请，待审批状态

**初始状态**
- 本月配额：2天
- 已用天数：0天
- 可用天数：2天

**第1次申请：1月5日，1天休假**
```
申请前检查：
  已用天数：0天
  可用天数：2天
  申请天数：1天
  结果：✅ 通过，创建休假记录

数据库记录：
  - 1月5日：休假1天（pending）
  - 已用天数：0 → 1天
  - 可用天数：2 → 1天
```

**第2次申请：1月6日，1天休假**
```
申请前检查：
  已用天数：1天（第1次申请，虽然待审批但已计入）
  可用天数：1天
  申请天数：1天
  结果：✅ 通过，创建休假记录

数据库记录：
  - 1月5日：休假1天（pending）
  - 1月6日：休假1天（pending）
  - 已用天数：1 → 2天
  - 可用天数：1 → 0天
```

**第3次申请：1月7日，1天休假**
```
申请前检查：
  已用天数：2天（前两次申请都已计入）
  可用天数：0天
  申请天数：1天
  结果：⚠️ 超出可用天数，弹出确认对话框

确认对话框：
  本月可用休假天数：0天
  申请天数：1天
  将自动创建：
  - 休假 0 天（带薪，不扣工资）
  - 请假 1 天（无薪，扣工资）

  具体日期分配：
  1月7日:请假

  是否继续？

数据库记录：
  - 1月5日：休假1天（pending）
  - 1月6日：休假1天（pending）
  - 1月7日：请假1天（pending）← 自动转换为请假
```

**初始状态**

  - 1月5日：休假1天（rejected）
- 已用天数：0天（拒绝的记录不计入）

**申请：1月6日，1天休假**

## 代码实现

### 后端查询
```javascript
// attendance.repository.js
async getUserUsedMonthlyLeaveDays(employeeId, year, month) {
  const query = `
    SELECT COALESCE(SUM(monthly_leave_days), 0) as used_days
    FROM ${this.tableName}
    WHERE employee_id = ?
      AND record_type = 'monthly_leave'
      AND status IN ('pending', 'approved')  // ← 包括待审批
      AND YEAR(record_date) = ?
      AND MONTH(record_date) = ?
  `;
  const result = await this.executeQuery(query, [employeeId, year, month]);
  return result[0]?.used_days || 0;
}
```

## 用户体验优势

1. **实时反馈**：申请时立即看到可用天数变化
2. **防止超额**：不会让用户申请超过配额的带薪休假
3. **透明清晰**：确认对话框显示具体哪些天是休假，哪些天是请假
4. **灵活调整**：如需调整，可以撤回待审批的申请

## 注意事项

1. **待审批占用配额**：申请提交后立即占用配额，即使还未审批
2. **拒绝不占用**：被拒绝的申请不会占用配额
3. **撤回释放配额**：如果撤回申请，配额会相应释放
4. **按日期拆分**：多天申请会按日期分别创建记录
