# 批发/代划拨功能实施总结

## 📋 功能说明

本系统现在支持两种库存调拨场景：

### 场景1：批发给同行（Wholesale）
- **产生货款**：计算到供应商的应付账款
- **计算利润**：批发价 - 进货成本
- **业务类型**：`wholesale`
- **手机状态**：`peer_transfer`

### 场景2：代供应商划拨（Supplier Proxy）
- **不产生货款**：不计算到供应商的应付账款
- **不计算利润**：仅记录划拨价格
- **业务类型**：`supplier_proxy`
- **手机状态**：`supplier_proxy`

## ✅ 已完成的工作

### 1. 数据库设计
**文件位置**：`/backend/database/migrations/add_peer_transfer_fields_v2.sql`

**新增表**：
- `wholesale_records` - 批发/划拨记录表

**修改表**：
- `phones` - 增加批发/划拨相关字段
- `customers` - 增加客户类型字段（retail/wholesale/both）

### 2. 后端API接口
**文件位置**：
- `/backend/src/controllers/transfer.controller.js`
- `/backend/src/services/transfer.service.js`
- `/backend/src/routes/transfers.js`

**API端点**：
- `POST /api/transfers/wholesale` - 批发给同行
- `POST /api/transfers/proxy` - 代供应商划拨
- `GET /api/transfers/records` - 获取记录列表
- `GET /api/transfers/statistics` - 获取统计数据

### 3. 前端组件
**文件位置**：`/frontend/src/components/WholesaleModal.vue`

- 批发对话框组件
- 代划拨对话框组件

### 步骤2：在后端注册新路由

编辑 `/backend/src/app.js`，添加批发/划拨路由：

```javascript
// 在其他路由导入后添加
const transfersRouter = require('./routes/transfers');

// 在其他路由注册后添加
app.use('/api/transfers', unifiedAuth, transfersRouter);
```

### 步骤3：重启后端服务

```bash
cd /Users/imac/Desktop/webtset/TF2025/backend
npm start
```

### 步骤4：在前端销售页面集成

在 `/frontend/src/views/sales/SalesView.vue` 中：

1. **导入组件**：
```vue
<script setup>
import WholesaleModal from '@/components/WholesaleModal.vue'
</script>
```

2. **添加状态**：
```javascript
const showWholesaleModal = ref(false)
const wholesaleMode = ref('wholesale') // 'wholesale' 或 'proxy'
```

3. **添加按钮**（在批量操作区域）：
```vue
<el-button
  v-if="selectedPhones.length > 0"
  type="primary"
  @click="showWholesaleModal = true; wholesaleMode = 'wholesale'"
>
  <i class="fas fa-boxes"></i>
  批发给同行
</el-button>

<el-button
  v-if="selectedPhones.length > 0"
  type="warning"
  @click="showWholesaleModal = true; wholesaleMode = 'proxy'"
>
  <i class="fas fa-exchange-alt"></i>
  代供应商划拨
</el-button>
```

4. **添加对话框**：
```vue
<WholesaleModal
  v-model:visible="showWholesaleModal"
  :mode="wholesaleMode"
  :phone-ids="selectedPhones.map(p => p.id)"
  :phones="selectedPhones"
  @success="handleTransferSuccess"
/>
```

5. **添加成功回调**：
```javascript
const handleTransferSuccess = () => {
  clearBatchSelection()
  loadAvailablePhones()
  ElMessage.success('操作成功')
}
```

## 🧪 测试场景

### 场景1：批发给同行
1. 选择一台在库手机
2. 点击"批发给同行"
3. 选择客户（同行类型）
4. 输入批发价格（如5050）
5. 确认批发
6. 检查：手机状态变为 `peer_transfer`，产生利润记录

### 场景2：代供应商划拨
1. 选择一台在库手机
2. 点击"代供应商划拨"
3. 选择供应商
4. 输入划拨价格（可选）
5. 确认划拨
6. 检查：手机状态变为 `supplier_proxy`，不产生利润
