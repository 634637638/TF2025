# 批发功能自动创建销售记录

## 功能说明

批发给同行功能现在会自动创建销售记录，与正常零售销售保持一致的数据追踪。

## 业务逻辑

### 批发给同行（Wholesale）
- ✅ 产生货款，计入供应商应付
- ✅ 计算利润（批发价 - 入库价）
- ✅ **自动创建销售记录**（sale_type = 'wholesale'）
- ✅ 手机状态变为 "同行调货"（peer_transfer）

### 代供应商划拨（Proxy Transfer）
- ❌ 不产生货款
- ❌ 不计算利润
- ❌ **不创建销售记录**（仅记录划拨）
- ✅ 手机状态变为 "代划拨"（supplier_proxy）

## 技术实现

### 后端修改

**文件**: `/backend/src/services/transfer.service.js`

在 `wholesaleToPeer` 方法中，批发记录创建成功后添加销售记录：

```javascript
// 创建销售记录（批发模式需要创建销售记录）
const saleInsertResult = await connection.query(
  `INSERT INTO sales
    (phone_id, customer_id, sale_type, price, cost, operator_id,
     payment_method, invoice_number, remarks, sale_date, store_id)
  VALUES (?, ?, 'wholesale', ?, ?, ?, NULL, NULL, ?, NOW(), ?)`,
  [
    phone_id,
    final_customer_id,
    price,
    final_purchase_cost,
    operator_id,
    remarks || null,
    store_id
  ]
);

const sale_id = saleInsertResult[0].insertId;

// 更新手机的销售记录ID
await connection.query(
  'UPDATE phones SET sale_id = ? WHERE id = ?',
  [sale_id, phone_id]
);
```

## 数据库变化

### sales 表
批发操作会自动插入记录：
- `sale_type`: 'wholesale'（标识为批发销售）
- `phone_id`: 手机ID
- `customer_id`: 客户ID（同行客户）
- `price`: 批发价格
- `cost`: 入库成本
- `operator_id`: 操作员ID
- `store_id`: 销售门店ID
- `sale_date`: 销售日期

### phones 表
- `status`: 'peer_transfer'（同行调货）
- `sale_id`: 关联的销售记录ID
- `business_type`: 'wholesale'（业务类型）

## 测试场景

### 1. 批发给同行
1. 选择一台在库手机
2. 点击"批发给同行"
3. 选择客户（同行类型）
4. 选择销售门店
5. 输入销售员姓名
6. 设置批发价格
7. 确认批发
8. 验证：
   - 手机状态变为"同行调货"
   - 创建了批发记录（wholesale_records）
   - 创建了销售记录（sales，sale_type='wholesale'）
   - 计算了利润

### 2. 代供应商划拨
1. 选择一台在库手机
2. 点击"代划拨"
3. 选择供应商
4. 设置划拨价格
5. 确认划拨
6. 验证：
   - 手机状态变为"代划拨"
   - 创建了划拨记录（wholesale_records）
   - **没有**创建销售记录
   - 没有计算利润
