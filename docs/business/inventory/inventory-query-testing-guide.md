# 在库查询功能测试指南

## 功能概述

价格查询页面新增了"在库查询"功能，允许用户输入密码验证后，双击型号查看该型号在各店铺的**在库商品**（按在库时间排序，优先显示在库最久的商品）。

## 核心功能

**只显示在库商品**：模态框弹出的数据都是状态为"在库"且为"新机"的商品，已售出的商品不会显示。

## 当前状态说明

⚠️ **重要提示**：
- 后端 API 尚未部署到生产服务器
- 目前使用 **localStorage** 存储设置
- 密码验证使用 **本地验证 + 默认密码** 兜底方案
- 在库数据展示使用 **模拟数据** 作为演示

## 功能特点

1. **密码验证**：在搜索框输入密码后自动验证
2. **双击查看**：验证成功后，双击表格中的型号列即可查看在库信息
3. **本地缓存**：使用 localStorage 临时存储设置
4. **独立密码**：可与屏幕锁定密码分开设置
5. **友好提示**：清楚告知用户当前使用的验证方式和数据来源

## 测试步骤

### 1. 设置查询密码

访问系统设置页面：
```
http://localhost:5176/system
```

找到"批发报价查询"设置卡片，设置查询密码。

**注意**：
- 如果不设置密码，可以使用默认密码 `123456`
- 密码将保存到浏览器 localStorage（本地存储）
- 后端部署后，设置会同步到服务器

### 2. 访问价格查询页面

```
http://localhost:5176/price-query
```

### 3. 输入密码验证

在搜索框中输入密码，按回车或点击搜索按钮。

#### 验证流程：

1. **优先使用本地配置密码**：
   - 如果在系统设置中配置了独立密码，直接验证
   - 成功提示：✅ "密码验证成功！现在可以双击型号查看在库信息"
   - 搜索框显示绿色"已验证"标签

2. **尝试后端验证**：
   - 如果没有配置本地密码，尝试连接后端 API
   - 如果后端返回 404，继续下一步

3. **默认密码兜底**：
   - 使用默认密码 `123456` 进行验证
   - 成功提示：✅ "密码验证成功！现在可以双击型号查看在库信息"
   - 附加提示：⚠️ "提示：请在系统设置中配置独立的查询密码"

### 4. 双击型号查看在库信息

验证成功后，在价格列表中找到任意型号，双击型号单元格。

#### 数据加载流程：

1. **优先请求后端 API**：
   - 调用 `/api/v1/phones/longest-inventory`
   - 传递品牌、型号、颜色、内存等参数

2. **后端失败时使用模拟数据**：
   - 提示：⚠️ "后端服务未连接，显示模拟数据"
   - 显示 3 条模拟数据：
     - 总店：在库约 XX 天
     - 广场店：在库约 XX 天
     - 三小店：在库约 XX 天

### 5. 查看在库信息

弹出对话框显示：
- **产品信息**：品牌、型号、颜色、内存
- **在库商品列表**（**仅显示在库商品**，按在库天数排序）：
  - 排名（1-3 名有特殊标识）
  - 店铺名称
  - IMEI 码
  - 入库日期
  - 在库天数（颜色标记）
    - 🟢 绿色：< 10 天（正常）
    - 🟡 黄色：10-19 天（注意）
    - 🟠 橙色：20-29 天（警告）
    - 🔴 红色：≥ 30 天（严重）

**重要说明**：
- ✅ **显示**：状态为"在库"且为"新机"的商品
- ❌ **不显示**：已售出、旧机、或其他非在库状态的商品

## 常见问题

### Q: 双击没有反应？
**A**: 请确保已通过密码验证，搜索框应显示绿色"已验证"标签。

### Q: 密码验证失败？
**A**:
1. 检查是否在系统设置中配置了密码
2. 确认密码输入正确
3. 尝试使用默认密码 `123456`
4. 查看浏览器控制台是否有错误信息

### Q: 弹窗显示"后端服务未连接，显示模拟数据"？
**A**: 这是正常的，说明后端 API 尚未部署。系统会显示模拟数据供演示使用。

### Q: 模拟数据会显示什么内容？
**A**: 显示 3 条示例数据：
- 总店：IMEI `4901542032375185`，入库日期 2024-01-15
- 广场店：IMEI `4901542032375186`，入库日期 2024-02-01
- 三小店：IMEI `4901542032375187`，入库日期 2024-02-20

### Q: 如何启用真实数据？
**A**: 需要等待后端部署以下路由：
1. `/api/v1/phones/longest-inventory` - 在库查询 API
2. `/api/v1/screen-lock/verify-inventory-query` - 密码验证 API

## 开发备注

### 本地存储键名
```javascript
localStorage.setItem('screenLockSettings', JSON.stringify({
  backgroundType: 'default',
  imageUrl: '',
  videoUrl: '',
  title: '屏幕已锁定',
  message: '请输入密码解锁',
  inventoryQueryPassword: 'your-password' // 查询密码
}))
```

### 验证流程
```
用户输入密码
    ↓
检查 localStorage 配置密码
    ↓ (无配置或失败)
尝试后端 API 验证
    ↓ (404)
默认密码 123456 兜底
    ↓
验证成功，启用双击功能
```

### 数据加载流程
```
用户双击型号
    ↓
请求后端 API (/phones/longest-inventory)
    ↓ (404)
显示模拟数据 + 友好提示
    ↓
渲染在库信息列表
```
