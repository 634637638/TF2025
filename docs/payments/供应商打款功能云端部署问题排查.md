# 供应商打款功能 - 云端部署问题排查和修复

## 问题描述

**现象**: 
- ✅ 本地环境：货款结算功能正常
- ❌ 云端环境：点击"销售管理 → 货款结算"无反应

**根本原因**: 
云端数据库缺少供应商打款功能的菜单、模块和权限配置。

---

## 问题分析

### 1. 代码层面检查

前端组件位置：
```
frontend/src/views/payments/SupplierPhonePaymentsView.vue
```

路由配置（`frontend/src/router/index.ts`）:
```typescript
{
  path: 'payments',
  name: 'Payments',
  component: () => import('@/views/payments/SupplierPhonePaymentsView.vue'),
  meta: {
    title: '供应商打款',
    icon: 'fas fa-money-bill-wave'
  }
}
```

组件使用的权限码：
- 模块 key: `supplier-payments`
- 查看权限：`payments_supplierphonepaymentsview:view`

### 2. 数据库配置要求

需要在云端数据库中创建以下数据：

#### 2.1 模块（modules 表）
```sql
`key` = 'supplier-payments'
name = '供应商打款'
```

#### 2.2 菜单（menus 表）
```sql
name = 'supplier-payments'
title = '供应商打款'
url = '/payments'
```

#### 2.3 权限（permissions 表）
需要创建以下权限：
- `supplier-payments:view` - 查看打款页面
- `supplier-payments:read` - 查看打款记录
- `supplier-payments:create` - 创建打款申请
- `supplier-payments:approve` - 审批打款申请
- `supplier-payments:confirm` - 确认付款完成
- `supplier-payments:cancel` - 取消打款申请
- `supplier-payments:export` - 导出打款记录

#### 2.4 关联表
- `menu_modules`: 菜单和模块关联
- `menu_permissions`: 菜单权限关联
- `role_permissions`: 角色权限关联

## 解决方案

### 方案一：通过菜单管理界面检查（推荐）

#### 步骤 1: 登录云端后台
访问：http://v6.cn9527.cn:30000

使用管理员账号登录。

#### 步骤 2: 打开浏览器开发者工具
- Chrome/Edge: 按 F12
- Firefox: 按 F12
- Safari: 开发 -> 显示网页检查器

#### 步骤 3: 切换到 Network (网络) 标签页
确保已开启记录网络请求。

#### 步骤 4: 访问菜单管理
在系统中找到"菜单管理"或"系统设置 → 菜单管理"并点击进入。

#### 步骤 5: 查找菜单列表 API
在网络请求中查找：
```
GET /api/menus
```

#### 步骤 6: 检查响应数据
点击该请求，查看响应内容（Response），搜索以下关键词：
- ✓ `supplier-payments`
- ✓ `供应商打款`
- ✓ `货款结算`
- ✓ `payments`

**如果找不到**: 说明云端数据库缺少菜单配置，需要执行 SQL 脚本。

### 方案二：直接在云端服务器执行 SQL（需要 SSH 权限）

如果你有云端服务器的 SSH 访问权限，可以直接执行 SQL 脚本。

#### 步骤 1: SSH 登录云端服务器
```bash
ssh root@v6.cn9527.cn
```

#### 步骤 2: 进入后端目录
```bash
cd /www/wwwroot/v6.cn9527.cn/backend
```

#### 步骤 3: 执行数据库迁移脚本
```bash
mysql -uTF2025 -pTF2025 TF2025 < database/migrations/add_supplier_payment_menu_complete.sql
```

或者分步查询检查：

**检查模块是否存在**:
```bash
mysql -uTF2025 -pTF2025 TF2025 -e "SELECT id, \`key\`, name FROM modules WHERE \`key\` = 'supplier-payments' OR name LIKE '%打款%';"
```

**检查菜单是否存在**:
```bash
mysql -uTF2025 -pTF2025 TF2025 -e "SELECT id, name, title, url FROM menus WHERE url LIKE '%payments%' OR title LIKE '%打款%';"
```

**检查权限是否存在**:
```bash
mysql -uTF2025 -pTF2025 TF2025 -e "SELECT code, name FROM permissions WHERE code LIKE '%payment%' OR code LIKE '%supplier%';"
```

如果以上查询结果为空，说明需要执行完整的数据迁移脚本。

## 常见问题

### Q1: 菜单出现了，但点击提示无权限
**原因**: 当前用户角色没有访问权限。

**解决**: 
1. 登录超级管理员账号
2. 进入角色管理
3. 为对应角色分配以下权限：
   - `supplier-payments:view`
   - `supplier-payments:read`

### Q2: 页面加载后显示空白或错误
**原因**: 后端 API 接口未正确配置。

**解决**:
1. 检查后端是否部署了最新的代码
2. 查看后端日志：`/www/wwwroot/v6.cn9527.cn/backend/logs/error-*.log`
3. 确认 API 路由配置正确

### Q3: 本地可以，云端不可以
**原因**: 本地和云端数据库不同步。

**解决**: 
- 在云端执行与本地相同的数据库迁移脚本
- 脚本位置：`backend/database/migrations/add_supplier_payment_menu_complete.sql`
