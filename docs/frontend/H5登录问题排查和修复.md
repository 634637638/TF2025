# H5用户认证系统问题排查和修复

## 问题描述
用户在访问 `http://localhost:5176/m/login?redirect=/m/my` 时遇到登录错误：`Request failed with status code 400`

## 已修复的问题

### 1. API路径错误 ✅ 已修复
**问题**：前端API调用路径缺少 `/public` 前缀
- 前端调用：`/auth/login`
- 后端路由：`/api/public/auth/login`

**修复**：
更新 `/frontend/src/api/auth.ts`，所有认证API路径添加 `/public` 前缀：
```typescript
export function login(data: LoginParams) {
  return publicApi.request<AuthResponse>({
    url: '/public/auth/login',  // 添加了 /public 前缀
    method: 'post',
    data
  })
}
```

同样修复了其他API：
- `/public/auth/register`
- `/public/auth/me`
- `/public/auth/orders`
- `/public/auth/sales`
- `/public/auth/logout`

### 2. publicApi导入错误 ✅ 已修复
**问题**：`auth.ts` 中使用了命名导入，但 `shop-public.ts` 是默认导出

**修复**：
```typescript
// 修改前
import { publicApi } from './shop-public'

// 修改后
import publicApi from './shop-public'
```

### 3. 错误处理改进 ✅ 已改进
**改进**：在 `shop-public.ts` 响应拦截器中添加了更详细的错误处理
