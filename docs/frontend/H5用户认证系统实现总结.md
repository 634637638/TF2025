# H5用户认证系统实现总结

## 概述
完成了H5移动端商城的用户认证系统，包括用户注册、登录、个人信息管理、订单查询和购买记录查看功能。

## 已完成的工作

### 1. 后端实现

#### 1.1 认证API（已集成到shop-public.js）
在 `/backend/src/routes/shop-public.js` 中添加了以下认证相关接口：

- `POST /api/public/auth/register` - 用户注册
- `POST /api/public/auth/login` - 用户登录
- `GET /api/public/auth/me` - 获取当前用户信息
- `GET /api/public/auth/orders` - 获取用户订单（H5_orders表）
- `GET /api/public/auth/sales` - 获取用户购买记录（sales表，通过customer_id关联）
- `POST /api/public/auth/logout` - 用户登出

#### 1.2 数据库迁移
创建了 `/backend/migrations/add_customer_auth.sql`：
- 在customers表中添加password字段（VARCHAR(255)，存储bcrypt加密的密码）
- 创建customer_tokens表（存储用户登录token，30天有效期）
- 添加avatar字段（如果不存在）

**执行迁移SQL**:
```bash
mysql -u your_user -p your_database < backend/migrations/add_customer_auth.sql
```

## 数据库关系说明

### 客户与订单关系
```
customers (客户表)
├── id (主键)
├── name (姓名)
├── phone (手机号)
├── password (密码，bcrypt加密)
└── source (来源，H5用户标记为'H5用户')

customer_tokens (登录token表)
├── customer_id (外键 → customers.id)
├── token (UUID)
└── expires_at (过期时间)

H5_orders (H5商城订单表)
├── customer_phone (关联customers.phone)
└── ... (订单信息)

sales (销售记录表)
├── customer_id (外键 → customers.id)
└── ... (销售信息)
```

### 重要说明
1. **H5_orders**: 通过 `customer_phone` 字段与客户表关联
2. **sales**: 通过 `customer_id` 外键与客户表关联
3. 用户登录后可以通过token获取customer_id，进而查询相关的销售记录

## 使用流程

### 新用户注册流程
1. 访问 `/m/my` 页面
2. 点击"注册账号"
3. 填写姓名、手机号、密码
4. 注册成功后自动登录并跳转到"我的"页面

### 用户登录流程
1. 访问 `/m/my` 页面
2. 点击"立即登录"
3. 输入手机号和密码
4. 登录成功后跳转回"我的"页面

### 查看订单和购买记录
1. 登录后访问"我的"页面
2. 点击"我的订单"查看H5商城订单
3. 点击"购买记录"查看历史销售记录

## 遗留用户处理

对于已在sales表中存在但未设置密码的客户：
- 登录时会提示"该账户未设置密码，请联系管理员重置密码"
- 管理员可以通过后台设置密码（需要开发相关功能）

## 安全特性

1. **密码加密**: 使用bcrypt加密，salt rounds = 10
2. **Token管理**: UUID token，30天有效期
3. **请求认证**: 使用Bearer Token在Authorization header中传递
4. **输入验证**: 前后端双重验证（手机号格式、密码长度等）
