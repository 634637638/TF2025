# TF2025 统一文档结构

## 📚 文档组织

项目文档统一放在根目录 `docs/` 下，按以下结构组织：

```
docs/
├── 00-INDEX.md              # 总索引
├── api/                     # API 文档
├── backend/                 # 后端文档
├── frontend/                # 前端文档
├── database/                # 数据库文档
├── deployment/              # 部署文档
├── development/             # 开发文档
├── security/                # 安全文档
├── permissions/             # 权限文档
├── payments/                # 支付文档
├── sync/                    # 同步文档
├── business/                # 业务文档
├── components/              # 组件文档
├── guides/                  # 指南文档
└── archive/                 # 归档文档
```

- 接口定义

### 2. 后端文档 (`backend/`)
- 架构设计
- 中间件说明
- 服务层设计
- 迁移计划

### 3. 前端文档 (`frontend/`)

- 页面结构
- 状态管理
- 路由配置

### 4. 数据库文档 (`database/`)
- 表结构
- 字段说明
- 索引设计
- 迁移脚本

### 5. 部署文档 (`deployment/`)
- 环境配置

- 云端部署

### 6. 开发文档 (`development/`)
- 开发规范
- 代码风格
- 调试技巧
- 性能优化

### 7. 安全文档 (`security/`)
- 安全加固
- 漏洞修复
- 权限控制
- 数据加密

### 8. 权限文档 (`permissions/`)
- 权限系统
- 字段权限
- 菜单权限
- 角色管理

### 9. 支付文档 (`payments/`)
- 支付流程
- 供应商支付
- 账单管理

### 10. 同步文档 (`sync/`)
- 数据同步
- 智能匹配
- 远程同步

## 📖 文档编写规范

### 1. 文件命名
- 使用大写 + 下划线：`API_STANDARDS.md`
- 或使用小写 + 连字符：`api-standards.md`

### 2. 文档格式
- 使用 Markdown 格式
- 包含目录（TOC）
- 添加代码示例
- 使用表格对比

### 3. 文档更新
- 重大变更更新文档
- 删除文档移至 archive/
- 保留文档版本历史
