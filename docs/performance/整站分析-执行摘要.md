## 🔴 严重问题（立即修复）

### P0-1: 数据库查询性能差
**影响**: 多个接口响应时间 > 500ms  
**原因**: 
- 缺少索引
- 使用 `SELECT *`
- OFFSET分页慢

**解决方案**: 
```bash
mysql -u root -p TF2025 < scripts/optimize_indexes.sql
```

**预期提升**: 查询速度 **5-10倍**

---

### P0-2: 缓存策略缺陷
**影响**: PM2集群模式下缓存无效，内存泄漏  
**原因**: 
- 使用Map作为缓存（进程隔离）
- 定时器未清理

**解决方案**: 
1. 短期: 修复定时器泄漏
2. 长期: 引入Redis

**预期提升**: 缓存命中率 **95%+**

---

### P0-3: 前端组件过大
**影响**: 维护困难，Bug率高  
**问题文件**:
- SalesView.vue: 10,431行
- SalaryView.vue: 7,743行
- PermissionsView.vue: 7,519行

**解决方案**: 拆分为 composables + components

**预期提升**: 可维护性 **300%**

---

### P0-4: 数据库连接池配置不当
**影响**: 连接不稳定，频繁重建  
**当前配置**:
- connectionLimit: 10 (太小)
- idleTimeout: 60s (太短)

**解决方案**: 
```javascript
connectionLimit: 20,
idleTimeout: 300000,
connectTimeout: 30000
```

**预期提升**: 稳定性 **80%**

---

### P0-5: API超时过长
**影响**: 用户体验差  
**当前**: 120秒  
**建议**: 30秒（特殊接口单独配置）

## 🟡 中等问题（近期优化）

### P1-1: 权限系统复杂
**问题**: 800+行映射表，每次查库  
**方案**: 动态规则 + 权限缓存  
**提升**: 检查速度 **5倍**

### P1-2: 日志过度记录
**问题**: 生产环境日志过多  
**方案**: 分级日志 + 自动轮转  
**节省**: 磁盘空间 **70%**

### P1-3: 图片处理阻塞
**问题**: HEIC转换同步进行  
**方案**: Sharp异步处理 + CDN  
**提升**: 上传速度 **3倍**

### P1-4: 前端请求优化不足
**问题**: 防抖500ms过长  
**方案**: 减少到300ms + 并行加载  
**提升**: 响应速度 **40%**

### P1-5: 缺少慢查询监控
**问题**: 性能问题发现滞后  
**方案**: 实时监控 + 告警  
**提升**: 发现时间从天级→分钟级
